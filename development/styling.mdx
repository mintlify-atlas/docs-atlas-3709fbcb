Astro Services uses Tailwind CSS v4 for styling, providing a modern, utility-first approach to design with a dark theme optimized for streaming services.

## Tailwind CSS 4 Setup

The project uses Tailwind CSS v4 with the new Vite plugin architecture.

### Installation

```json package.json
{
  "dependencies": {
    "@tailwindcss/vite": "^4.2.1",
    "tailwindcss": "^4.2.1"
  }
}
```

### Vite Configuration

Tailwind is integrated directly into Vite:

```js vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react-swc'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

### CSS Import

The main stylesheet imports Tailwind directives:

```css src/index.css
@import "tailwindcss";
```

<Note>
  Tailwind CSS v4 uses a simpler import syntax compared to v3. No need for separate `@tailwind` directives.
</Note>

### ServiceCard Component

The service card showcases advanced styling with hover effects:

```jsx components/ServiceCard.jsx
<div className="group relative flex flex-col overflow-hidden rounded-2xl bg-gray-900 shadow-lg transition-all duration-300 hover:shadow-2xl hover:shadow-indigo-500/20 hover:-translate-y-1">
  {/* Image with hover scale */}
  <div className="relative h-48 overflow-hidden">
    <img
      className="h-full w-full object-cover transition-transform duration-500 group-hover:scale-110"
      src={imagen}
      alt={titulo}
    />
    
    {/* Discount Badge */}
    {descuento > 0 && (
      <span className="absolute top-3 right-3 rounded-full bg-emerald-500 px-3 py-1 text-xs font-bold text-white shadow-md">
        -{descuento}%
      </span>
    )}
  </div>

  {/* Content */}
  <div className="flex flex-1 flex-col gap-3 p-5">
    <h3 className="text-lg font-bold text-white">{titulo}</h3>
    <p className="flex-1 text-sm leading-relaxed text-gray-400">{descripcion}</p>

    {/* Price Display */}
    <div className="flex items-end gap-2 pt-2">
      <span className="text-2xl font-extrabold text-indigo-400">S/.{precioFinal}</span>
      {descuento > 0 && (
        <span className="mb-0.5 text-sm text-gray-500 line-through">S/.{precio.toFixed(2)}</span>
      )}
    </div>

    {/* CTA Button */}
    <button className="mt-2 w-full rounded-xl bg-indigo-600 py-2.5 text-sm font-semibold text-white transition-colors duration-200 hover:bg-indigo-500 active:bg-indigo-700">
      Contratar
    </button>
  </div>
</div>
```

#### Advanced Techniques:

1. **Group Hover**: `group` class enables child elements to respond to parent hover
   ```jsx
   <div className="group">
     <img className="group-hover:scale-110" />
   </div>
   ```

2. **Multiple Transforms**: Combining shadow, scale, and translate
   ```css
   hover:shadow-2xl hover:shadow-indigo-500/20 hover:-translate-y-1
   ```

3. **Opacity Colors**: Using `/20` for 20% opacity
   ```css
   bg-red-950/50  /* 50% opacity red-950 */
   shadow-indigo-500/20  /* 20% opacity shadow */
   ```

4. **Conditional Classes**: Rendering styles based on data
   ```jsx
   {descuento > 0 && <span className="...">-{descuento}%</span>}
   ```

## Responsive Design

### Mobile-First Approach

Tailwind encourages mobile-first design. Styles apply to mobile by default, then scale up:

```jsx
{/* Hidden on mobile, visible on desktop */}
<div className="hidden md:block">
  Desktop Navigation
</div>

{/* Visible on mobile, hidden on desktop */}
<div className="md:hidden">
  Mobile Menu
</div>

{/* Responsive padding */}
<main className="px-6 py-12">
  {/* Content */}
</main>
```

### Hamburger Menu Animation

The mobile menu uses transition utilities:

```jsx
<button onClick={() => setMenuOpen(!menuOpen)}>
  <span className={`block h-0.5 w-6 bg-gray-400 transition-all duration-300 ${
    menuOpen ? 'translate-y-2 rotate-45' : ''
  }`} />
  <span className={`block h-0.5 w-6 bg-gray-400 transition-all duration-300 ${
    menuOpen ? 'opacity-0' : ''
  }`} />
  <span className={`block h-0.5 w-6 bg-gray-400 transition-all duration-300 ${
    menuOpen ? '-translate-y-2 -rotate-45' : ''
  }`} />
</button>

{/* Expandable menu */}
<div className={`overflow-hidden transition-all duration-300 ${
  menuOpen ? 'max-h-40' : 'max-h-0'
}`}>
  {/* Menu content */}
</div>
```

<Accordion title="Use Semantic Class Names">
  Group related utilities logically:

## Performance Considerations

### JIT Compilation

Tailwind CSS v4 uses Just-in-Time (JIT) compilation by default:

- Only generates CSS for classes you actually use
- Smaller production bundle sizes
- Faster build times

### CSS Purging

In production builds, unused CSS is automatically removed. Ensure dynamic classes are safelisted if needed:

```js
// If you need dynamic classes:
const bgColor = `bg-${color}-500`; // ❌ Won't be detected

// Use complete class names:
const bgColor = color === 'blue' ? 'bg-blue-500' : 'bg-gray-500'; // ✅
```
