This guide will help you get the Astro Services application up and running on your local machine quickly. You'll have both the frontend and backend running with sample data.

## Prerequisites

Before you begin, ensure you have:

- **Node.js v18 or higher** installed ([download here](https://nodejs.org/))
- **Git** for cloning the repository
- A **MongoDB Atlas** account (free tier works) or local MongoDB instance
- Basic familiarity with the terminal

<Steps>

### Clone the repository

First, clone the Astro Services repository and navigate into the project directory:

```bash
git clone https://github.com/luisllatas-dev/proyecto_streaming.git
cd Proyecto_Astro
```

### Set up the backend

Install the backend dependencies:

```bash
cd astro_backend
npm install
```

Create a `.env` file in the `astro_backend` directory with your MongoDB connection string:

```env
PORT=5000
MONGDBD_URI=mongodb+srv://<usuario>:<password>@<cluster>.mongodb.net/astroservicesdb
```

<Note>
Replace `<usuario>`, `<password>`, and `<cluster>` with your actual MongoDB Atlas credentials. You can find your connection string in the MongoDB Atlas dashboard under "Connect".
</Note>

### Start the backend server

Run the backend in development mode with hot reload:

```bash
npm run dev
```

You should see output similar to:

```
MongoDB conectado: cluster0-shard-00-01.xxxxx.mongodb.net
Servidor corriendo en http://localhost:5000
```

<Tip>
The backend uses Node's `--watch` flag to automatically restart when you make changes to the code.
</Tip>

### Seed the database

Open a new terminal window and populate the database with sample services:

```bash
cd astro_backend
npm run seed
```

This will insert 6 sample streaming services (Netflix, Disney+, HBO Max, Spotify, IPTV, and Amazon Prime Video) into your database:

```
MongoDB conectado para seeding...
Colección de servicios limpiada.
6 servicios insertados correctamente.
Conexión cerrada. Seed completado.
```

### Set up the frontend

In another terminal window, install the frontend dependencies:

```bash
cd astro_services
npm install
```

This installs React 19, Vite 7, and Tailwind CSS 4.

### Start the frontend

Launch the development server:

```bash
npm run dev
```

You should see:

```
  VITE v7.3.1  ready in 324 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

### Access the application

Open your browser and navigate to:

```
http://localhost:5173
```

You should see a grid of service cards displaying the 6 streaming services you seeded earlier.

</Steps>

## Verify the setup

Test that the API is working correctly:

```bash
curl http://localhost:5000/api/health
```

Fetch all services:

```bash
curl http://localhost:5000/api/services
```

You should receive a JSON array with all the seeded services.

## What's running?

You now have:

- **Backend API** at `http://localhost:5000` serving REST endpoints
- **Frontend application** at `http://localhost:5173` with hot module replacement
- **MongoDB database** connected and populated with sample data

## Troubleshooting

<Warning>
If you see `Error de conexión a MongoDB`, double-check your `MONGODB_URI` in the `.env` file and ensure your MongoDB Atlas IP whitelist includes your current IP address (or use `0.0.0.0/0` for development).
</Warning>

**Port already in use?** If port 5000 or 5173 is occupied, you can change them:

- Backend: Modify `PORT` in your `.env` file
- Frontend: Vite will automatically try the next available port

**Connection refused on frontend?** Make sure the backend is running on port 5000. The frontend's API client expects the backend at `http://localhost:5000/api` in development mode (see `astro_services/src/services/api.js:3`).
