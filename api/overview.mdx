## Base URL

The API runs on port 5000 by default:

```
http://localhost:5000/api
```

All API endpoints are prefixed with `/api`.

## Available Endpoints

The Astro Services API provides the following endpoints:

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/services` | Retrieve all services |
| `GET` | `/services/:id` | Retrieve a single service by ID |
| `POST` | `/services` | Create a new service |
| `PUT` | `/services/:id` | Update an existing service |
| `DELETE` | `/services/:id` | Delete a service |
| `GET` | `/health` | Health check endpoint |

## Authentication

The current API does not require authentication. All endpoints are publicly accessible.

## Request Format

All POST and PUT requests must include a `Content-Type: application/json` header and send data in JSON format.

```bash
curl -X POST http://localhost:5000/api/services \
  -H "Content-Type: application/json" \
  -d '{"titulo": "Service Name", ...}'
```

## Error Handling

The API uses standard HTTP status codes:

- **200** - OK: Request succeeded
- **201** - Created: Resource created successfully
- **400** - Bad Request: Invalid input or validation error
- **404** - Not Found: Resource not found
- **500** - Internal Server Error: Server-side error

All error responses include:
- `message`: Human-readable error description (in Spanish)
- `error`: Technical error details


## CORS Configuration

The API supports Cross-Origin Resource Sharing (CORS) with the following configuration:

- **Origin**: Configurable via `FRONTEND_URL` environment variable (defaults to `*`)
- **Credentials**: Enabled

## Rate Limiting

The API does not currently implement rate limiting. All endpoints can be called without restrictions.

## Sorting and Pagination

### Sorting

The `GET /api/services` endpoint returns services sorted by creation date in descending order (newest first).

### Pagination

Pagination is not currently implemented. All services are returned in a single response.
