Check that Node.js and npm are properly installed:

## Clone the repository

Get the source code from GitHub:

```bash
git clone https://github.com/luisllatas-dev/proyecto_streaming.git
cd Proyecto_Astro
```

The project structure looks like this:

```
Proyecto_Astro/
├── astro_services/          # Frontend (React + Vite)
├── astro_backend/           # Backend (Express + MongoDB)
└── docker-compose.yml       # Docker orchestration
```

## Backend installation

### Install dependencies

Navigate to the backend directory and install packages:

<CodeGroup>
```bash npm
cd astro_backend
npm install
```

```bash yarn
cd astro_backend
yarn install
```

```bash pnpm
cd astro_backend
pnpm install
```
</CodeGroup>

This installs the following key dependencies:

| Package | Version | Purpose |
|---------|---------|----------|
| `express` | ^4.21.2 | Web framework for REST API |
| `mongoose` | ^8.9.5 | MongoDB ODM |
| `dotenv` | ^16.4.7 | Environment variable management |
| `cors` | ^2.8.5 | Cross-origin resource sharing |

### Configure environment variables

Create a `.env` file in the `astro_backend` directory:

```bash
touch .env
```

Add the following configuration:

```env
# Server Configuration
PORT=5000

# Database Configuration
MONGDB_URI=mongodb+srv://<usuario>:<password>@<cluster>.mongodb.net/astroservicesdb
```

<Note>
**MongoDB Atlas Setup:**
1. Create a free account at [mongodb.com/atlas](https://www.mongodb.com/atlas)
2. Create a new cluster (free M0 tier works perfectly)
3. Click "Connect" and choose "Connect your application"
4. Copy the connection string and replace `<usuario>` and `<password>`
5. Whitelist your IP address (or use `0.0.0.0/0` for development)
</Note>

#### Local MongoDB alternative

If you prefer running MongoDB locally:

```env
PORT=5000
MONGDB_URI=mongodb://localhost:27017/astro_services
```

<Tip>
The backend uses Google DNS servers (`8.8.8.8`, `8.8.4.4`) to resolve MongoDB Atlas SRV records, which improves connection reliability (see `astro_backend/src/config/db.js:5`).
</Tip>

### Configure API endpoint (optional)

By default, the frontend connects to `http://localhost:5000/api` in development. If your backend runs on a different port, create a `.env` file in `astro_services`:

```env
VITE_API_URL=http://localhost:5000/api
```

The API client at `astro_services/src/services/api.js:3` automatically switches between development and production URLs:

```javascript
const API_BASE_URL = import.meta.env.VITE_API_URL || 
  (import.meta.env.PROD ? '/api' : 'http://localhost:5000/api');
```

## Database seeding

Populate your database with sample streaming services:

```bash

npm run seed
```

This script (`astro_backend/src/seed.js:53`) will:

2. Clear existing services (if any)
3. Insert 6 sample services:

## Docker installation (alternative)

If you prefer using Docker, you can run the entire stack with Docker Compose:

```bash
docker-compose up --build
```

This launches three containers:

| Container | Port | Description |
|-----------|------|-------------|
| `astro_mongo` | 27017 | MongoDB 7 database |
| `astro_backend` | 5000 | Express API server |
| `astro_frontend` | 80 | React application |

Access the application at `http://localhost` (port 80).

<Warning>
Docker Compose uses a local MongoDB instance at `mongodb://mongo:27017/astro_services`, not MongoDB Atlas. You don't need a `.env` file when using Docker.
</Warning>

## Troubleshooting

### MongoDB connection issues

**Error: "Error de conexión a MongoDB"**

- Verify your `MONGODB_URI` is correct in `.env`
- Check MongoDB Atlas IP whitelist includes your IP
- Ensure your MongoDB Atlas user has read/write permissions
- Try using Google DNS by adding to your `.env`:
  ```env
  NODE_OPTIONS="--dns-result-order=ipv4first"
  ```

### Port conflicts

**Error: "EADDRINUSE: address already in use"**

If port 5000 is occupied:

```bash
# Find process using port 5000
lsof -i :5000

# Kill the process
kill -9 <PID>

# Or change the port in .env
PORT=5001
```

### Module not found errors

**Error: "Cannot find module"**

Ensure you ran `npm install` in both directories:

```bash
# Reinstall backend
cd astro_backend
rm -rf node_modules package-lock.json
npm install

# Reinstall frontend
cd ../astro_services
rm -rf node_modules package-lock.json
npm install
```

### Frontend can't connect to backend

**Error: "Failed to fetch"**

- Ensure backend is running on port 5000
- Check CORS is enabled (it's configured in the backend by default)
- Verify `VITE_API_URL` if you set a custom backend URL
- Check browser console for specific error messages

### Node version issues

**Error: "Unsupported engine"**

Upgrade Node.js to v18 or higher:

```bash
# Check current version
node --version

# Install nvm (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install and use Node 18
nvm install 18
nvm use 18
```
